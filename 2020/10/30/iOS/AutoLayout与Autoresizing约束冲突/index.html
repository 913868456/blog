AutoLayout与Autoresizing约束冲突

问题：

经常看到使用SnapKit进行布局的时候，明明各个约束都写好了，但是进入页面的时候，日志打印显示约束冲突，如下：

"<NSAutoresizingMaskLayoutConstraint:0x283cf6620 h=--& v=--& ShiHuiGroup.PerformanceHeaderView:0x10c128b60.width == 0   (active)>",
"<SnapKit.LayoutConstraint:0x28383e700@PerformanceHeaderView.swift#110 UILabel:0x10c128d40.left == UIView:0x10c1291b0.left + 15.0>",
"<SnapKit.LayoutConstraint:0x28383e7c0@PerformanceHeaderView.swift#110 UILabel:0x10c128d40.right == UIView:0x10c1291b0.right - 15.0>"

该问题说 Autoresizing定义的约束为 width =0，且该约束处于激活状态；
但是SnapiKit却设置的是Leading和Traling，有宽度的，因此会报冲突

原因：

UIView 的translatesAutoresizingMaskIntoConstraints 属性默认为ture，将Autoresizing翻译成的约束与AutoLayout的约束产生冲突

官方文档解释：

translatesAutoresizingMaskIntoConstraints

A Boolean value that determines whether the view’s autoresizing mask is translated into Auto Layout constraints.

该属性是决定view的自动尺寸适配掩码是否翻译成 AutoLayout 的约束

Note that the autoresizing mask constraints fully specify the view’s size and position; therefore, you cannot add additional constraints to modify this size or position without introducing conflicts. 

注意 自动尺寸适配掩码已经完全指定了view的尺寸和位置，因此，你不需要添加额外的约束去修改该视图的尺寸和位置,从而避免引入冲突


If you want to use Auto Layout to dynamically calculate the size and position of your view, you must set this property to false, and then provide a non ambiguous, nonconflicting set of constraints for the view.

如果想要使用AutoLayout去动态计算视图的位置和尺寸，必须将该属性置为false,并且给该视图提供 一个明确的，无冲突的约束集合

By default, the property is set to true for any view you programmatically create. If you add views in Interface Builder, the system automatically sets this property to false.

默认情况下，该属性是true,如果你的视图在Interface Builder 中，系统自动将改属性置为false

解决方法：

将该视图的属性设置为false

小技巧：

如果一个视图内的多个子视图报该问题，那么只需要把其父视图的该属性关闭即可
